# get_repository_guidance

Get comprehensive repository configuration including note guidance, tag restrictions, and documentation standards.

## Overview

The `get_repository_guidance` tool provides detailed information about repository-specific configuration, documentation standards, and best practices for using A24z Memory effectively.

<RequestExample>
```json
{
  "path": "/src/components"
}
```
</RequestExample>

## Parameters

| Parameter | Type   | Required | Description                                |
| --------- | ------ | -------- | ------------------------------------------ |
| `path`    | string | Yes      | File or directory path to get guidance for |

## Response Format

### Complete Response

````json
{
  "success": true,
  "repositoryPath": "/Users/project/my-app",
  "configuration": {
    "noteGuidance": {
      "creationGuidelines": "Always include code examples for technical notes. Use markdown formatting.",
      "requiredFields": ["note", "directoryPath", "tags"],
      "tagRequirements": "Minimum 1 tag, maximum 5 tags per note",
      "contentStandards": "Use clear, actionable language. Include context and examples."
    },
    "tagRestrictions": {
      "allowedTags": ["react", "typescript", "api", "testing", "architecture"],
      "restrictedTags": ["deprecated", "temp", "fixme"],
      "requiredTags": ["typescript"],
      "customTags": {
        "micro-frontend": "Module federation specific patterns",
        "shared-library": "Cross-application shared code"
      }
    },
    "documentationStandards": {
      "markdownStyle": "Use GitHub Flavored Markdown with code blocks",
      "codeExamples": "Include TypeScript examples with proper typing",
      "fileReferences": "Use absolute paths from repository root",
      "languageStyle": "Technical, concise, actionable"
    }
  },
  "template": {
    "noteTemplate": "# Problem\n\n[Describe the problem]\n\n# Solution\n\n[Describe the solution]\n\n# Example\n\n```typescript\n// Code example\n```\n\n# Related Files\n\n- `/src/components/Example.tsx`\n- `/src/hooks/useExample.ts`",
    "decisionTemplate": "# Decision\n\n[What was decided]\n\n# Context\n\n[Why this decision was needed]\n\n# Options Considered\n\n- [Option 1]\n- [Option 2]\n\n# Chosen Solution\n\n[Why this option was chosen]\n\n# Consequences\n\n[Impact of this decision]"
  }
}
````

### Basic Response

```json
{
  "success": true,
  "repositoryPath": "/Users/project/my-app",
  "configuration": {
    "noteGuidance": {
      "creationGuidelines": "Document important decisions and patterns",
      "requiredFields": ["note", "directoryPath", "tags"],
      "contentStandards": "Use clear, actionable language"
    },
    "tagRestrictions": null,
    "documentationStandards": null
  },
  "template": null
}
```

## Configuration Sources

The tool reads configuration from multiple sources in order of priority:

### 1. Repository Configuration File

`.a24z-memory.json` in the repository root:

```json
{
  "noteGuidance": {
    "creationGuidelines": "Include code examples and context",
    "requiredFields": ["note", "directoryPath", "tags", "type"],
    "contentStandards": "Use markdown with code blocks"
  },
  "tagRestrictions": {
    "allowedTags": ["react", "typescript", "api", "testing"],
    "requiredTags": ["typescript"],
    "customTags": {
      "micro-frontend": "Module federation patterns"
    }
  }
}
```

### 2. Default Templates

If no custom configuration exists, the tool provides sensible defaults:

```json
{
  "noteGuidance": {
    "creationGuidelines": "Document important insights, decisions, and patterns",
    "requiredFields": ["note", "directoryPath", "tags"],
    "contentStandards": "Use clear, actionable language with examples"
  }
}
```

## Guidance Categories

### Note Creation Guidelines

**Purpose**: Help maintain quality and consistency in note creation.

**Examples**:

- "Always include concrete code examples for technical notes"
- "Use the active voice and present tense"
- "Include file paths and line numbers when relevant"
- "Document both the problem and the solution"

### Tag Requirements

**Purpose**: Ensure consistent categorization and discoverability.

**Examples**:

- Required tags: `["typescript"]` (for TypeScript projects)
- Allowed tags: `["react", "api", "testing", "architecture"]`
- Restricted tags: `["todo", "fixme", "deprecated"]`

### Documentation Standards

**Purpose**: Maintain consistent documentation quality across the team.

**Examples**:

- Use GitHub Flavored Markdown
- Include TypeScript examples with proper types
- Reference files using repository-rooted paths
- Use technical but accessible language

## Template System

### Note Templates

The tool provides structured templates for different types of notes:

#### Decision Template

```
# Decision
[What was decided and why]

# Context
[Background and requirements]

# Options Considered
- Option 1: [Pros and cons]
- Option 2: [Pros and cons]

# Chosen Solution
[Why this option was selected]

# Implementation
[How it was implemented]

# Consequences
[Expected impact and follow-up actions]
```

#### Pattern Template

````
# Problem
[What problem does this pattern solve?]

# Solution
[How to implement the pattern]

# Example
```typescript
// Code example showing the pattern
````

# When to Use

[Appropriate use cases]

# Related Patterns

[Similar patterns and alternatives]

````

### Customization

Repositories can define custom templates in `.a24z-memory.json`:

```json
{
  "templates": {
    "bugFix": "# Bug\n\n[Describe the bug]\n\n# Root Cause\n\n[What caused it]\n\n# Fix\n\n[How it was fixed]\n\n# Testing\n\n[How to verify]",
    "apiDesign": "# Endpoint\n\n[API endpoint details]\n\n# Request/Response\n\n[Schema details]\n\n# Implementation\n\n[Code example]"
  }
}
````

## Error Handling

### Repository Not Found

```json
{
  "success": false,
  "error": "No Git repository found at the specified path",
  "suggestion": "Ensure the path is within a Git repository"
}
```

### Configuration Error

```json
{
  "success": false,
  "error": "Invalid repository configuration",
  "details": "Syntax error in .a24z-memory.json at line 15",
  "suggestion": "Fix the JSON syntax error in your configuration file"
}
```

### Permission Denied

```json
{
  "success": false,
  "error": "Permission denied accessing repository configuration",
  "suggestion": "Check file permissions for .a24z-memory.json"
}
```

## Best Practices

### Configuration Management

1. **Version Control**: Keep `.a24z-memory.json` in Git
2. **Team Review**: Review configuration changes with the team
3. **Documentation**: Document custom tags and templates
4. **Regular Updates**: Update standards as the project evolves

### Template Usage

1. **Adapt Templates**: Modify provided templates to fit your needs
2. **Add Examples**: Include concrete code examples in templates
3. **Team Input**: Get team feedback on template usefulness
4. **Version Updates**: Update templates as coding standards evolve
