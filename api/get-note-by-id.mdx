# get_repository_note

Retrieve a specific note by its unique ID for detailed information. This tool provides complete note details including content, metadata, anchors, and timestamps.

## Overview

The `get_repository_note` tool allows you to fetch a specific note from the knowledge base using its unique identifier. This is useful when you need detailed information about a particular note, such as when reviewing content, checking metadata, or examining related files.

<RequestExample>
```json
{
  "noteId": "note-1734567890123-abc123def",
  "directoryPath": "/Users/project/my-app"
}
```
</RequestExample>

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `noteId` | string | Yes | Unique note ID (format: note-timestamp-hash) |
| `directoryPath` | string | Yes | Absolute path to the git repository root directory |

## Response Format

### Successful Retrieval
```json
{
  "success": true,
  "message": "# Note ID: note-1734567890123-abc123def\n\n**Type:** decision\n**Confidence:** high\n**Tags:** architecture, api, authentication\n**Created:** 2024-01-15T10:30:00.000Z\n\n## Anchors\n- /src/api/authService.ts\n- /src/components/LoginForm.tsx\n- /src/hooks/useAuth.ts\n\n## Content\nWe decided to implement JWT-based authentication instead of session-based authentication for the following reasons:\n\n1. **Scalability**: JWTs can be validated without server-side state\n2. **Mobile Support**: Better support for mobile applications\n3. **Microservices**: Easier to implement across multiple services\n\nThe implementation uses RS256 signing with a 1-hour token expiration and 7-day refresh token expiration.\n\n## Metadata\n- **alternatives_considered:** Session-based auth, API key authentication\n- **implementation_date:** 2024-01-15\n- **review_date:** 2024-06-15"
}
```

### Note Not Found
```json
{
  "success": false,
  "error": "Note with ID \"note-1734567890123-abc123def\" not found in repository \"/Users/project/my-app\". Please verify the note ID is correct."
}
```

## Note Structure

### Core Information
- **Note ID**: Unique identifier in format `note-timestamp-hash`
- **Type**: Note classification (decision, pattern, gotcha, explanation)
- **Confidence**: Creator's confidence level (high, medium, low)
- **Tags**: Semantic tags for categorization
- **Created**: ISO timestamp of creation

### Content Sections
- **Anchors**: List of related file/directory paths
- **Content**: Main note content in Markdown format
- **Metadata**: Optional structured data (if provided)

### Metadata Examples

**Decision Notes**:
```json
{
  "alternatives_considered": ["Option A", "Option B"],
  "rationale": "Detailed explanation of why this choice was made",
  "implementation_date": "2024-01-15",
  "review_date": "2024-06-15"
}
```

**Pattern Notes**:
```json
{
  "context": "When to apply this pattern",
  "prerequisites": "Required conditions",
  "testing_approach": "How to test implementations",
  "related_patterns": ["pattern1", "pattern2"]
}
```

**Gotcha Notes**:
```json
{
  "symptoms": "How to identify this issue",
  "root_cause": "What causes this problem",
  "solution": "How to fix it",
  "prevention": "How to avoid it in the future"
}
```

## Error Handling

### Repository Not Found
```json
{
  "success": false,
  "error": "directoryPath is not within a git repository: \"/Users/project/my-app\". Please provide a path within a git repository."
}
```

### Invalid Path
```json
{
  "success": false,
  "error": "directoryPath does not exist: \"/Users/project/my-app\". Please provide a valid absolute path to an existing directory."
}
```

### Permission Denied
```json
{
  "success": false,
  "error": "Permission denied accessing repository notes"
}
```

### Invalid Note ID Format
```json
{
  "success": false,
  "error": "Invalid note ID format. Expected format: note-timestamp-hash"
}
```

- Note IDs are generated automatically and are unique
- IDs follow the format: `note-timestamp-hash`
- IDs are stable and don't change over time
- IDs can be shared and referenced safely

- Ideal for programmatic access and automation
